supercontent

아이들이 즐기는, 교육과정에 맞는 학습 콘텐츠.

의도를 읽고, 학년 밴드가 무엇에 반응하는지 짚고, 알맞은 매체로 라우팅해, 결정적 교육-적합성 게이트를 통과한 콘텐츠를 만듭니다. 분위기 점검이 아니라 게이트가 판정합니다. 교실 밖 직장인 업무 문서(보고서·제안서·PDF·PPT·한글 HWPX)도 DOCS 모드로 생성·읽기·변환합니다 - Windows/macOS 호환.

왜 이게 평범한 프롬프트보다 나은가

교육과정 먼저, 재미는 그 다음.

재미는 전달 방식이지 목적지가 아닙니다. 성취기준에서 벗어난 콘텐츠는 아무리 재미있어도 결함입니다.

의도 기반 라우팅

요청을 11개 모드 중 하나로 분류하고, 학년 밴드에 맞는 매체와 인지 부하를 고릅니다. 하나의 기본 틀이 아니라 의도가 매체를 정합니다.

트렌드 인지, 트렌드 추종 아님

밴드가 무엇에 반응하는지 짚되, 브리프에 도움이 되는 것만 적용합니다. 트렌드 읽기는 날짜와 함께 기록됩니다.

적합성은 눈대중이 아니라 강제

빌더는 절대 스스로 승인하지 않습니다. edu-critic이 게이트를 돌리고 아무것도 편집하지 않습니다.

하나의 집 스타일이 아니다

의도가 모드와 매체를 정한다.

신호를 읽고 알맞은 매체로 보냅니다. 모든 빌드 모드는 자유롭게 만들고 같은 게이트로 검증됩니다.

DIALS는 claims.md에 선언: FUN_INTENSITY / COGNITIVE_LOAD / SCAFFOLDING (각 low|med|high). 밴드 기본값 - 초저: FUN high / LOAD low / SCAFFOLD high. 고: FUN med / LOAD med-high / SCAFFOLD low-med. 단 DOCS(직장인 업무 문서)는 dials와 교육 게이트 대신 doc-gate.sh(맞춤법 + 사실 무결성 + 이모지/PII)를 씁니다 - curriculum-claims.json 없이.

눈대중이 아니라 결정적 게이트

예측 가능한 실패는 grep 가능하다.

스크립트가 판정을 계산하므로 조용한 결함이 통과할 수 없습니다. 빌더는 절대 스스로 채점하지 않습니다.

역할 분리 루프

빌드는 자기 작업을 채점하지 않는다.

저자 독립성. 단일 작품은 인라인으로 역할을 바꾸며, 자료 배치는 에이전트로 오케스트레이션합니다.

  1. Read학년 / 과목 / 성취기준 / 학습목표를 브리프 한 줄로.
  2. Pulse밴드가 반응하는 것을 짚고, 날짜와 함께 다이얼 선언.
  3. Direction한 포맷 패밀리 + 매체. EXPLORE면 사용자가 고른 뒤 빌드.
  4. Buildpedagogy-core에 맞춰. 실제/생성 미디어만, 사실은 출처.
  5. Critique독립 실행. edu-gate.sh를 돌리고 모든 위반 기록, 편집 없음.
  6. Verify위반마다 최소 수정, 초록될 때까지 재실행. 최대 3회.

자기완결, 우아한 폴백. 도구가 없으면 Claude-Code-only / 로컬 경로로 강등하고 대체를 기록합니다.

품질 우선, 폴백 포함. 학생 데이터가 관여하면 로컬/자체 호스팅(Supertonic TTS, presenton+Ollama)을 선호하고, 외부 전송은 동의 하드스톱입니다.

참조 레포 (역량 + 출처): youtube-autopilot, OpenMontage, supertonic-tts, Math-To-Manim, youtube-transcript-api, presenton, ppt-master. DOCS 모드: python-docx, python-pptx, pdfplumber, pypdf, python-hwpx, WeasyPrint, LibreOffice+H2Orestart. 자매 스킬 패턴: superdesign.
