# explore - divergent format directions, build nothing

EXPLORE mode. The user has a learning objective but no fixed medium ("어떤 형식이 좋을까", "방향 잡아줘"). Offer 2-4 genuinely different format directions + one recommendation. **Build nothing** - the user picks, then you run the build loop.

## First, lock the objective

- Confirm 학년 band, 과목, and the 학습목표 (what the learner can DO). One question max if it is unclear.
- Pulse trends for the band (`trend-research.md`) or use the snapshot - directions should reflect what engages this band in 2026.

## Generate 2-4 directions that actually differ

Differ on the LEARNING MECHANIC, not just the medium label. Pull from the mode catalog (VIDEO / MANIM / POSTER / WEB / SLIDES / GAME / AUDIO), but each direction must teach the objective a distinct way:

- by **passive narrative** (VIDEO/AUDIO story) vs **active manipulation** (WEB widget/GAME) vs **static reference** (POSTER) vs **guided sequence** (SLIDES) vs **formal visualization** (MANIM).
- Reject near-duplicates: "영상 A" and "영상 B" are not two directions.

Per direction, one tight block:

- **이름** - format family + medium.
- **메커니즘** - how it teaches the objective (the active beat).
- **밴드 적합** - why it fits the band's dials (FUN/LOAD/SCAFFOLD).
- **비용/위험** - tool needs, fallback, any safety/저작권 flag.

## Recommend one

- Pick the simplest direction that best lands the objective for the band - cite the dials and the trend read.
- Name the runner-up and when you would choose it instead.

## Boundary

No scripts, no assets, no vault build, no gate run. Output is the directions + recommendation only. When the user picks, enter that medium's build loop from step 3 (Direction) carrying the locked objective and band forward.
